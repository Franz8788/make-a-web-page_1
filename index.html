Coding: An introduction

Part 1

The internet in general

The internet is the largest global computer network. How does it work? Basically, we need the following elements for it to run: Servers, the internet, the HTTP protocol (Hypertext transfer protocol), and a browser to read the files that are transferred.

HTML and CSS

HTML is the language used on the internet. The abbreviation stands for Hypertext Markup Language meaning that here is not only text, but also rules how the text is structured. CSS (Cascading Style Sheets) concern the presentation of a document, which can be changed by changing the CSS.

HTML elements

HTML elements consist of contents and tags. Tags are made up of tag names and angled brackets surrounding the tag names. Often the contents are wrapped by opening tags and closing tags. But there are also void tags such as <br> (Break). <br> stands alone and needs no closing tag later in the text. Other examples for tags are: <b> (bold, opening tag), </b> (bold, closing tag), <em> and </em> for italicized parts, <p> and </p> for opening and closing a paragraph.

Inline and block elements

In general, inline elements are flowing with the text of the document (for example: <img>), block elements have an invisible box around them (for example: <p> and </p>). But we were assured that for beginners it is not necessary to understand the differences in every detail.

Links and images

Links are inline elements. Same goes for images. An example for an inactive link is (included here for clarification): <a href=”http://www.udacity.com”>inactive content</a>. An example for an active link is: active content. href represents an attribute. In the case of images the attribute is not href, but src.

Attention: Do not mix up src and href.

Part 2

Front end development

Being inherently visual front end development is an excellent starting point for learning programming. HTML and CSS have different purposes. If we compare front end development to building a house, HTML refers to the walls (the structure) and CSS to the style (for example the color of the walls).

Tree-like structure of web documents

When programmers speak about the DOM (Domain Object Model), they mean the tree-like structure of web pages. The terminology of DOM elements is similar to family terminology with terms such as parent element, child element or uncle element. A very good way to explore web documents are the developer tools of a browser like Chrome.

Search for all the boxes

Programmers with some experience find it easy to see a web document as a set of boxes or to be more correct nested boxes.

Useful tools for programmers

Scratchpad.io can be used for smaller projects. Otherwise Sublime Text 2 is recommended. Of course, every simple text editor can be used, too, but work will progress at a slower pace.

Avoid repetition

Programmers want to write code which is as parsimonious as possible. That means repetition must be avoided. One of the advantages of less repetition is fewer mistakes. If programmers have to fix fewer things, there will be less room for error.

What to remember in any case, part 2

What is the visual shape of all HTML elements?

The shape is always rectangular, even if it is not obvious at first sight.

If you see a whiteboard with a tree-like structure of connected boxes, what is it?

It is the DOM (Domain Object Model) of a web page.
